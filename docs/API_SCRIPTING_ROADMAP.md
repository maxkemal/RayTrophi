## 1. Vizyon ve Model Kararı

RayTrophi'ye Blender-tarzı script/otomasyon desteği eklenecek. **Seçilen model:**

- **Gömülü Python** (pybind11 embed) — script'ler uygulamanın kendi prosesinde, `rt` modülü üzerinden çalışır (Blender'daki `bpy` muadili).
- Tüm giriş noktaları (Python, ImGui konsolu, headless CLI, ileride IPC/JSON) **tek bir C++ fasadına** bağlanır: **`rtapi`** (RtApi).
- **Motor-DLL / C ABI modeli REDDEDİLDİ** (şimdilik): uygulama bir DCC, host edilebilir kütüphane değil. Dil bağımsızlığı ileride ABI'den değil, IPC/JSON protokolünden gelecek.

```
                    ┌────────────── RayTrophi.exe ───────────────┐
  Python script ──▶ │ gömülü Python (rt modülü)  ┐                │
  ImGui konsolu ──▶ │                            ├─▶ rtapi ──▶ SceneCommand,
  headless CLI  ──▶ │ arg parser (--script)      │   fasadı     SceneHistory,
  dış program   ──▶ │ IPC dinleyici (JSON, Faz4) ┘              NodeRegistry,
                    └────────────────────────────────────────────┘  Renderer...
```

## 2. Değişmez Kurallar (her faz için geçerli)

1. **Tek kapı:** Script/IPC/CLI hiçbir zaman iç sınıfları (Renderer, VulkanBackend, SceneData...) doğrudan çağırmaz; her şey `rtapi` fasadından geçer. pybind11 binding'leri SADECE `rtapi` fonksiyonlarını sarar.
2. **Handle = isim/ID, asla pointer:** API dışarıya `Triangle*`, `TriangleMesh*` vb. sızdırmaz. Objeler `nodeName` (string) ile adreslenir. Olmayan objeye erişim `Result::fail(...)` / Python'da exception döndürür.
3. **Main-thread kuralı:** Sahneyi değiştiren her API çağrısı main thread'de, frame'ler arasında çalışır. Başka thread'den gelen istekler `rtapi::enqueue()` kuyruğuna girer; `rtapi::drainMainThreadQueue()` ana döngüde frame başına bir kez çalışır (Main.cpp). Sahne yüklenirken (`scene_loading`) drain atlanır.
4. **Undo bedava:** Sahne mutasyonları mümkün olan her yerde mevcut `SceneCommand` + `SceneHistory` üzerinden yapılır (`cmd->execute(ctx)` + `history.record(...)` + `ProjectManager::markModified()`). Yeni API operasyonu = gerekiyorsa yeni SceneCommand türü.
5. **Sürümleme:** `rtapi::version()` semantik sürüm döner (şu an 0.1.0). 1.0'dan önce yüzey daralabilir; 1.0 sonrası kırılma = major artar. Yüzeyi DAR tut — fonksiyon eklemek kolay, geri almak imkânsız.
6. **Fasat başlığı hafif kalır:** `RtApi.h` ağır proje başlıklarını include etmez (forward declaration kullan; şu an sadece `Matrix4x4.h`). Ağır include'lar `RtApi.cpp`'de.
7. **Hata modeli:** C++ tarafında exception YOK; her mutasyon `rtapi::Result {ok, error}` döner. Python katmanı Result.ok=false'u exception'a çevirir.
